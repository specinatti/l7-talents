Portal L7 Talents — Versão 2.0 —

1. Visão Geral do Sistema

O L7 Talents é um portal de recrutamento e gestão de pessoas desenvolvido para conectar candidatos a oportunidades de emprego e empresas a talentos qualificados. A plataforma opera em modelo SaaS com acesso via browser, sem necessidade de instalação.

5. Segurança

5.1 Autenticação e Sessão

  • Autenticação via JWT (JSON Web Token) com expiração de 2 horas
  • Renovação automática de token a cada 90 minutos de uso ativo
  • Timeout de sessão por inatividade: 30 minutos com aviso 2 minutos antes
  • Senhas armazenadas com bcrypt (salt rounds: 10) — irreversível
  • Nenhuma senha é armazenada em texto puro em nenhum momento

5.2 Proteção de Rotas

  • Todas as rotas da API exigem token JWT válido (exceto login, cadastro e vagas públicas)
  • Controle de acesso por role: candidato, empregador, financeiro, admin
  • Tentativa de acesso a role diferente retorna HTTP 403

5.3 Rate Limiting

  • API geral: máximo 100 requisições por IP a cada 15 minutos
  • Login/Cadastro/Recuperação de senha: máximo 20 tentativas por hora por IP
  • Excesso retorna HTTP 429 com tempo de espera

5.4 Headers de Segurança

  • X-Content-Type-Options: nosniff — previne MIME sniffing
  • X-Frame-Options: DENY — previne clickjacking
  • X-XSS-Protection: 1; mode=block — proteção XSS no browser
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy — desabilita geolocalização, microfone e câmera
  • Header X-Powered-By removido (não expõe tecnologia)

5.5 Recuperação de Senha

  • Token de reset gerado com crypto.randomBytes(32) — 256 bits de entropia
  • Token expira em 1 hora e é invalidado após uso
  • Endpoint não revela se email existe (proteção contra enumeração)

5.6 Cache Busting

  • Assets CSS/JS recebem ?v=timestamp a cada deploy
  • Páginas HTML têm Cache-Control: no-cache
  • Garante que usuários sempre recebem a versão mais recente

6.2 Direitos dos titulares

  • Acesso: O usuário pode visualizar todos os seus dados no perfil
  • Correção: O usuário pode editar seus dados a qualquer momento
  • Exclusão: Conta pode ser desativada (campo ativo = false)
  • Portabilidade: Dados exportáveis mediante solicitação
  • Revogação: Usuário pode solicitar exclusão total via email

6.3 Armazenamento e retenção

  • Dados armazenados em servidor PostgreSQL gerenciado pelo Railway (infraestrutura AWS)
  • Backups automáticos diários (plano pago Railway)
  • Dados de candidaturas retidos enquanto a conta estiver ativa
  • Senhas nunca armazenadas — apenas hash irreversível

6.4 Compartilhamento de dados

  • Dados de candidatos são compartilhados apenas com empregadores para os quais o candidato se candidatou
  • Nenhum dado é vendido ou compartilhado com terceiros para fins comerciais
  • Infraestrutura: Railway.app (EUA) — coberto por cláusulas contratuais padrão

7. Funcionalidades do Portal

Área do Candidato

  • Cadastro e login com email/senha
  • Perfil completo: experiências, formações, habilidades, foto, currículo
  • Busca de vagas com filtros (área, modalidade, nível, contrato, salário)
  • Candidatura com carta de apresentação personalizada
  • Acompanhamento de status das candidaturas em tempo real
  • Chat com recrutadores via mensagens internas
  • Salvar vagas favoritas
  • Notificações de atualizações
  • Recuperação e troca de senha

Área do Empregador

  • Cadastro de empresa com perfil completo
  • CRUD completo de vagas (criar, editar, pausar, encerrar)
  • Gestão de candidatos por vaga com atualização de status
  • Visualização de perfis completos dos candidatos
  • Chat com candidatos
  • Dashboard com métricas de vagas e candidaturas
  • Feedback para candidatos

Área Financeiro/Gestão (comercial@l7talents.online)

  • KPIs: usuários totais, candidatos, empresas, vagas ativas, candidaturas
  • Gráficos de crescimento mensal (usuários e candidaturas)
  • Distribuição de vagas por área e status de candidaturas
  • Evolução diária de novos usuários (30 dias)
  • Ranking de empresas por vagas publicadas
  • Monitoramento em tempo real: uptime, latência do banco, memória
  • Log dos últimos erros do sistema

9. Procedimentos de Emergência

Site fora do ar

  1. Acesse railway.app → projeto l7-talents
  2. Verifique a aba Deployments — se o último deploy falhou, clique em Redeploy
  3. Verifique as Variables — confirme que DATABASE_URL e JWT_SECRET estão presentes

Usuário não consegue logar

  1. Verifique se o email está correto
  2. Use a função "Esqueci a senha" em /pages/esqueci-senha.html
  3. Se SMTP não estiver configurado, o link de reset aparece apenas nos logs do Railway

Redefinir senha de qualquer usuário (emergência)

Execute localmente com o .env configurado:

node -e "const b=require('bcryptjs'),{Pool}=require('pg');require('dotenv').config();const p=new Pool({connectionString:process.env.DATABASE_URL,ssl:{rejectUnauthorized:false}});b.hash('NOVA_SENHA',10).then(h=>p.query('UPDATE users SET password_hash=$1 WHERE email=$2',[h,'EMAIL']).then(()=>{console.log('ok');p.end()}))"
